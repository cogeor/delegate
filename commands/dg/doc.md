# /dg:doc - Generate Project Documentation

You generate and maintain documentation in `.delegate/doc/` that mirrors the project's directory structure.

## Output Structure

```
.delegate/doc/
├── README.md                    # Top-level: folder tree + "Last commit: {hash}"
├── src/
│   ├── README.md                # Documents src/ contents
│   ├── admin/
│   │   └── README.md            # Documents src/admin/ contents
│   └── utils/
│       └── README.md            # Documents src/utils/ contents
├── commands/
│   └── README.md                # Documents commands/ contents
└── ...                          # Mirrors every documented folder
```

**Key rule:** One `README.md` per folder. The doc tree mirrors the source tree exactly.

## Non-Negotiable Rules

1. **Mirror the codebase** — doc paths match source paths
2. **One README.md per folder** — each describes that folder's contents
3. **Incremental updates** — only re-document folders with changed files
4. **Track freshness** — top-level README.md records "Last commit: {hash}"
5. **Never modify source** — only write to `.delegate/doc/`
6. **Skip .delegate/, node_modules/, .git/, dist/, build/** — not documentable

## Pipeline

### Step 1: Check Last Documented Commit

1. Read `.delegate/doc/README.md`
2. Extract the `Last commit: {hash}` line
3. If no README.md or no hash → this is a **full generation** (document everything)
4. If hash found → this is an **incremental update**

### Step 2: Find Changed Files

**Full generation:**
```bash
git ls-files
```

**Incremental update:**
```bash
git log {last_hash}..HEAD --name-only --pretty=format:""
```

Filter out:
- Files in `.delegate/`, `node_modules/`, `.git/`, `dist/`, `build/`
- Deleted files (verify they still exist)
- Non-source files (binaries, images, lock files)

### Step 3: Group by Directory

Group changed files by their parent directory. Each unique directory = one README.md to generate/update.

Example:
```
Changed files:
  src/utils/auth.js
  src/utils/hash.js
  src/admin/dashboard.js
  commands/dg/doc.md

Directories to document:
  src/utils/
  src/admin/
  commands/dg/
```

### Step 4: Generate/Update README.md per Folder

For each directory that has changes:

1. List ALL files in that directory (not just changed ones)
2. Read each source file
3. Write `.delegate/doc/{path}/README.md`

**README.md format per folder:**

```markdown
# {folder-name}/

> Auto-generated by `/dg:doc`. Do not edit manually.

## Overview

{1-3 sentences: what this folder contains and its role in the project}

## Contents

### `{filename}`

{2-4 sentences: purpose, key exports/functions, how it fits in the module}

### `{filename}`

{2-4 sentences: purpose, key exports/functions, how it fits in the module}

## Dependencies

- `{import path}` — {what and why}
```

**Guidelines for folder READMEs:**
- Describe the folder's role first, then each file
- Focus on **why** things exist, not line-by-line **what**
- Note relationships between files in the folder
- If a subfolder exists, mention it briefly (its own README has details)
- Keep descriptions concise — 2-4 sentences per file max

### Step 5: Update Top-Level README.md

Write `.delegate/doc/README.md` with:

```markdown
# Project Documentation

> Auto-generated by `/dg:doc`. Do not edit manually.
>
> Last commit: {HEAD commit hash}
> Generated: {ISO timestamp}

## Structure

{tree of documented folders — use indented list}

## Modules

{one-line summary per top-level folder}
```

Generate the folder tree by listing all `README.md` files in `.delegate/doc/` and building the tree from their paths.

### Step 6: Report

```
Documentation updated
  Last commit: {hash}
  Folders documented: {count}
  New: {count}  Updated: {count}  Unchanged: {count}
```

## Edge Cases

- **Empty project:** Generate top-level README.md only
- **Deleted files:** Remove their entries from the parent folder's README.md. If a folder is entirely deleted, remove its README.md from `.delegate/doc/`
- **New folders:** Create new README.md
- **Renamed files:** Treated as delete + create
